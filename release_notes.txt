## BannerHub Lite v1.0.1

First stable on the BannerHub API.

> ⚠️ **URGENT — read before first launch.** If you see **"Failed to initialize the simulator"** when launching a game, your firmware is still on the old version. Open **Settings → Components → Firmware** and **manually update / install firmware 1.3.7**, then launch the game again. The auto-update prompt may not fire on every install, so do this manually if the error appears.

### What changed since v1.0.0

- **BannerHub API switch** — all 9 variants now point at `https://bannerhub-api.the412banner.workers.dev/` instead of the EmuReady worker. Components, firmware, and the in-app catalog are all served from the BannerHub Components release on this org.
- **Firmware bumped to 1.3.7** — the in-app firmware update prompt will offer `imagefs_137.zst` (171.9 MB). 5.x clients (BHL) and 6.0 clients (BannerHub 3.7.x) now receive the same firmware.
- **HID-mode launch dialog fix** — GameHub Lite 5.1.4 backports a touch/HID picker dialog from 5.3.5 that ships broken in 5.1.4 (transparent close button, unfocusable checkbox, no way to dismiss with controller). EmuReady's worker happened to 404 the underlying notice endpoint so the dialog never appeared; with BannerHub API the dialog would otherwise appear and block launches. The smali method that shows it (`IMappingServiceImpl.b()`) is rewritten to a no-op so games launch directly.

### Upgrade

Same package names as v1.0.0 — install over the existing variant; no uninstall needed.
